## Design Context

### Users
Espectro amplo: do aluno do 1º período de Direito ao advogado sênior. Usam o produto para criar, gerenciar e exportar documentos jurídicos (petições, contratos, modelos). Contexto de uso: desktop, ambiente de escritório ou faculdade, tarefas repetitivas que exigem agilidade.

### Brand Personality
**3 palavras**: Moderno, profissional, eficiente.
Inspiração de usabilidade: produtos Google (clareza, acessibilidade, sem fricção).
Anti-referência: sistemas jurídicos antiquados (TJ-SP style), interfaces sem personalidade que poderiam ser qualquer SaaS.
O produto deve se destacar no mercado jurídico brasileiro sem assustar usuários menos tech-savvy.

### Aesthetic Direction
- **Tema**: Light mode. Fundo bege quente já estabelecido — preservar.
- **Tom visual**: "Refined Clarity" — elegância discreta. Cada elemento deve parecer intencional. Nada supérfluo.
- **Animações**: Sutis e discretas, mas perceptíveis. Como física — elementos deslizam no lugar, não aparecem abruptamente. Durations curtos (120–350ms). Sem bounce, sem elastic.
- **Tipografia**: Manrope (UI) + Source Serif 4 (editor) são boas escolhas — manter. Hierarquia mais contrastada.
- **Cor**: Azul profissional (#2B5BB4 aprox.) como primária. Expandir sistema de sombras e usar OKLCH para tokens.

### Design Principles
1. **Clareza antes de beleza** — toda animação ou elemento visual deve ajudar a entender, não apenas decorar.
2. **Universal mas não genérico** — acessível para iniciantes, eficiente para experts. Nunca subestime nem sobrecarregue.
3. **Físico e intencional** — transições se comportam como objetos reais. Entrada suave, saída rápida.
4. **Hierarquia forte** — o que é mais importante deve ser imediatamente óbvio.
5. **Identidade própria** — um elemento visual que o usuário lembra. O bonsai, a paleta quente — preservar e ampliar.
